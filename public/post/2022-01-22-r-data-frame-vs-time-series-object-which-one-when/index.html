R Data Frame vs Time Series Object: Which one when?

January 22, 2022
R

Data frames are one of the fundamental building blocks of data analysis done in R. In some cases, though, time series objects offer more flexibility and other advantages when analyzing time-based data.

So when to choose time series rather data frames?

Time Series Object

xts is the main R package for working with time series. Datacamp has an excellent xts Cheat Sheet and very helpful supplementary info.

So I’m just going to hit some highlights with examples here.

## use quantmod pkg to get some stock prices as time series
ea <- getSymbols(Symbols='EA', auto.assign=FALSE)
head(ea)
##            EA.Open EA.High EA.Low EA.Close EA.Volume EA.Adjusted
## 2007-01-03    50.8    52.7   49.7     51.0   8206600        50.7
## 2007-01-04    51.7    54.0   51.5     53.7   7824100        53.3
## 2007-01-05    53.3    53.6   52.3     53.1   5806900        52.7
## 2007-01-08    53.2    53.2   52.0     52.9   2693500        52.5
## 2007-01-09    53.0    53.7   52.4     52.7   2146300        52.4
## 2007-01-10    52.6    53.9   52.3     53.7   2422700        53.3

Stock market prices are available in time series form via R quantmod package. As shown above, dates are row numbers rather than a date field as they would be in typical data frame. This has a variety of implications.
